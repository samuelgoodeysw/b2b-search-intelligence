# Search & Discovery Audit

## The problem

Customers often search before they browse.

If Magento search is slow, poorly configured, or disconnected from catalogue data, customers may struggle to find products that already exist in the catalogue.

For a B2B store, this can mean missed product discovery, repeated support friction, and avoidable pressure on sales or customer service teams.

## What the audit does

Search & Discovery Audit reviews Magento search and catalogue evidence to show where product discovery may be breaking down.

It identifies:

- searches where Magento returned zero results
- repeated failed-search terms
- catalogue evidence behind each issue
- likely root causes
- estimated demand value
- search configuration opportunities
- synonym and search-term opportunities
- product data and attribute issues
- visibility, stock, website, and category assignment issues
- optional GA4 low-engagement search signals where tracking data supports it

## Why audit first

The first step does not need a production module install.

We can run the audit against a sanitized Magento database dump or approved export, then produce a one-off report and walkthrough.

This gives the team useful evidence before making decisions about:

- catalogue cleanup
- search configuration
- synonyms
- layered navigation
- Adobe Live Search readiness
- future dashboard/module installation

## What the client gets

The audit deliverable includes:

- executive summary
- prioritized failed-search findings
- estimated demand value by search issue
- supporting Magento catalogue evidence
- likely diagnosis for each priority issue
- recommended review/fix steps
- search configuration opportunities
- catalogue hygiene opportunities
- optional GA4 low-engagement findings if available
- recommended follow-on roadmap

## Example findings

The audit can highlight issues such as:

- customers search for a product but Magento returns zero results
- the product exists but is not visible in search
- the product exists but is disabled or out of stock
- related products exist but key attributes are not searchable
- customers use different wording from the catalogue
- SKU, MPN, part number, or brand terms are not matching
- search terms need synonym or search-term setup
- layered navigation or attribute structure may be blocking product discovery
- GA4 shows search activity but little product engagement

## What this is not

This is not:

- a production module install
- a search engine replacement
- an Adobe Live Search replacement
- a guaranteed lost-revenue report
- an automatic catalogue changer
- a full layered navigation rebuild
- a promise of exact revenue recovery

The audit is evidence-led and diagnostic.

## Data needed

Preferred input:

- sanitized Magento database dump
- or approved extracts of relevant Magento tables
- optional GA4 export/access if low-engagement analysis is in scope
- known search complaints or priority product areas
- current search/platform context

## How it supports Adobe Live Search readiness

Before investing in Adobe Live Search or a wider search refactor, it helps to know whether the underlying catalogue and discovery structure is ready.

The audit can reveal:

- missing or weak product data
- non-searchable attributes
- poor synonym coverage
- inconsistent product naming
- category and layered navigation issues
- SKU/part-number matching gaps
- visibility, stock, website, and category assignment problems

This creates a practical roadmap before a larger search-platform decision.
